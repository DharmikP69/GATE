🟢 Most Scoring

  • Linear Algebra — eigenvalues appear every year; 3 problem types
  • Bayes' theorem — formula-driven, fast marks
  • Matrix rank/system of equations — predictable pattern

🔴 Difficult Areas

  • Eigenvalue decomposition + Cayley-Hamilton applications
  • Joint distributions and conditional expectation
  • Numerical method error propagation

⏱ Time-Consuming

  • Linear algebra (parametric systems, multiple unknowns)
  • Probability distribution problems (setup is slow)

⚠ Common Mistakes

  • Missing cases in Bayes' multi-event setup
  • Not verifying Gauss-Jordan pivot selection
  • Skipping Numerical Methods — usually 1 easy mark
PYQ Trend: Linear Algebra appears in 85%+ of GATE CS papers. Focus on 3 eigenvalue problem types: characteristic equation solving, Cayley-Hamilton theorem application, and diagonalisability check. Numerical Methods — low ROI, skip if time-constrained.

  • Inclusion-exclusion counting problems

🔴 Difficult Areas

🟢 Most Scoring

  • Stacks — infix/postfix always 1–2 marks
  • Hashing — collision resolution numericals
  • Heaps — extract/insert operation trace

🔴 Difficult Areas

  • Hashing — 3 open addressing methods with probe formulas

Trees are cross-subject — master them once and gain marks in DS, DBMS (B+ trees for indexing), OS (directory trees, free block lists), and Compiler Design (parse trees, ASTs). Highest ROI topic in the entire GATE CS syllabus.

🟢 Most Scoring

  • Recurrences — Master theorem applied directly every year
  • Huffman coding — greedy tree construction, guaranteed pattern
  • Sorting complexity table — comparison often 2 marks
  • Graph algorithms — Dijkstra/MST numericals always appear

🔴 Difficult Areas

  • DP — problem identification (recognising which subproblem to define) is harder than the recurrence itself
  • NP-completeness reductions — needs formal proof reasoning
  • Amortised analysis (aggregate, accounting, potential)

⏱ Time-Consuming

  • Dynamic Programming — 10+ distinct problem types, each unique
  • Graph algorithms — 8 algorithms with overlapping but distinct properties

⚠ Common Mistakes

  • Applying Master Theorem when f(n) has log factors — use extended case
  • Quicksort worst case O(n²) when sorted input + fixed pivot
  • Prim builds MST by cheapest edge to tree; Kruskal by globally cheapest non-cycle edge
  • NP-hard ≠ NP-complete (NP-complete must also be in NP)
GATE DP questions pick ONE specific problem — not generic DP design. Master these 8 by tracing manually: 0-1 knapsack, LCS, LIS, matrix chain, edit distance, coin change, subset sum, optimal BST. Tracing > coding for GATE.

🟢 Most Scoring

  • Number systems — binary/hex arithmetic is fast marks
  • K-map simplification — completely predictable question type
  • MUX implementation of Boolean functions — always tested

🔴 Difficult Areas

  • State machine design with don't-cares — multiple valid answers
  • JK flip-flop excitation table derivation under time pressure
  • Asynchronous sequential circuits — race hazards

⏱ Time-Consuming

  • Sequential circuits — Mealy/Moore state minimisation
  • 5-variable K-maps with don't-cares

⚠ Common Mistakes

  • K-map grouping — must use largest possible power-of-2 groups
  • IEEE 754: forgetting biased exponent (bias = 127 for single)
  • 2s complement overflow: + + = − or − − = + signals overflow
  • Mealy: output depends on state AND input; Moore: only on state
Number Systems + K-Maps + Combinational Circuits alone covers ~60% of DL marks and are the easiest to prepare. Sequential circuits have the highest difficulty but also highest marks potential. Flip-flop excitation table questions appear in nearly every GATE paper.

🟢 Most Scoring

  • Cache AMAT calculation — completely numerical, formulaic
  • Pipeline speedup and CPI — 2–3 marks every year
  • Addressing modes — fast once pattern is memorised
  • Cache mapping (direct-mapped tag/index/offset)

🔴 Difficult Areas

  • Data hazard forwarding — which stage sends to which
  • Multi-level cache AMAT (local vs global miss rate)
  • Microprogrammed control unit — horizontal vs vertical

⏱ Time-Consuming

  • Pipelining — hazard analysis requires full instruction trace
  • Cache — multiple mapping types × multiple write policies

⚠ Common Mistakes

  • Stall cycle count: RAW with no forwarding = (latency–1) stalls; with forwarding = 0 for most cases
  • Cache: tag bits = address bits − index bits − offset bits
  • Local miss rate ≠ global miss rate (GATE tests both)
  • Confusing pipeline throughput (instructions/cycle) with latency
Pipelining + Cache = 6–8 marks in every GATE paper — these two topics alone justify 45% of COA study time. Always practice AMAT with actual address calculations (not just formulas) and pipeline Gantt charts with data hazards.

⚠ Common Mistakes
